# 平衡态 MD vs SMD：VWF 2B 机制与方法学科普

> 面向汇报 / 非分子动力学背景读者。2026-06-24。
> 配套技术细节见 `docs/7A6O_SMD_LITERATURE_NOGO_2026-06-24.md`。

---

## 摘要（一段话）

VWF 的 2B 型突变本质是一个**力学现象**——"安全锁"变松了。要直接测它，理论金标准是
**SMD（用虚拟弹簧把锁拽开、量断裂力）**，但我们的 SMD 因**拽得太快（比实验快约一百万倍）**
+ **用错了突变类型**而完全失败（2B/2M 方向反了）。因此改用**平衡态 MD**：不施力，只观察
静止状态下"锁扣得牢不牢"（自抑制盐桥还在不在），作为 2B 的**间接但稳健的代理信号**。

## 一、必要的生物学背景：VWF 是一个"靠力开关的安全锁"

VWF（von Willebrand 因子）是血液里负责止血的大分子。它的工作方式很特别——**靠机械力开关**：

- **A1 结构域** = 抓血小板的"手"（通过结合血小板表面的 GPIbα 受体）。
- 平时血流平静时，一个叫 **AIM（autoinhibitory module，自抑制模块）** 的小帽子盖住 A1，
  **让它抓不到血小板** —— 这就是"安全锁"。
- 只有血管破损、血流**剪切力**骤增时，这股力把帽子"撕开"，A1 才暴露出来抓血小板止血。

设计精髓：**没力 → 锁住；有力 → 打开。**

两种致病突变正好对应锁的两种坏法：

| 类型 | 机制 | 通俗说法 | 临床后果 |
|---|---|---|---|
| **2B（功能增强 GOF）** | 帽子**太松**，不用多大力就被撕开 | 安全锁失灵，平时就乱抓血小板 | 血小板被消耗 → 反而出血 |
| **2M（功能丧失 LOF）** | A1 这只"手"本身坏了，抓不住 GPIbα | 锁正常，但手废了 | 抓不到 → 出血 |

**关键结论：2B 的本质是"撕开锁需要的力变小了"——它是个力学量。** 这句话决定了用什么方法测它。

---

## 二、两种分子模拟的本质区别

两者底层都是 **MD（molecular dynamics，分子动力学）**：用物理力场算每个原子的受力，
按牛顿方程一步步（每步约 2 飞秒 = 2×10⁻¹⁵ 秒）推进，看分子怎么动。
**区别只在于：有没有外加一只"手"去拽分子。**

（给做 AI 的同事的类比见下表。）

| | **平衡态 MD**（我们现在做的） | **SMD / 导向 MD**（之前失败的） |
|---|---|---|
| 做法 | 不加外力，体温(310K)下让分子自然热抖动 | 接一根虚拟弹簧，**恒速拽**，强行扒开 |
| 测什么 | 哪些结构/化学键**静止时稳定** | 扒开需要的**断裂力（rupture force）** |
| 物理状态 | **平衡态**（稳态分布） | **非平衡**（外部驱动） |
| AI 类比 | 从稳态分布**采样**（Boltzmann），只观察不干预 | 给系统加 ramp 输入、**测响应/做功**曲线 |
| 对 2B | 间接：看"锁扣得牢不牢"（盐桥占有率） | 直接：量"撕开锁要多大力"（最对题） |
| 难度 | 便宜、稳健 | 对题但**极难做对**（见第三节） |

### 为什么 2B 偏要这两种都考虑？

因为 **2B 的真身是"力"**，所以**最对题的金标准本应是 SMD**——直接量断裂力，2B 应该比 WT **更低**。
但 SMD 做对的门槛极高（见下），所以退而求其次用**平衡态盐桥**做代理：既然 2B 让锁松，
那也许静止时盐桥就先断了，不必真去拽。

## 三、我们的 SMD 为什么失败（根本原因，按重要性排序）

3 对照实测（slow025 慢速，断裂力单位 pN）：

| 变体 | 类型 | 实测断裂力 | 理论应有方向 |
|---|---|---|---|
| R1306W | **2B** | **1114 pN** ← 最高 | 应**最低** |
| WT | 野生型 | 1047 pN | 中间 |
| R1374H | **2M** | 982 pN ← 最低 | 应 ≈ WT |

判别指标 `AUC = 0.00`，意思是**完美反向**——2B 不但没最低，反而最高。
这说明结果整个反了，**不是生物学奇怪，是我们的实验设置错了**。根因两条：

### 根因①（主因）：拽得太快——快了约一百万倍

- 我们的拉伸速度 ≈ 0.25–1 nm/ns；实验（光镊 optical tweezers）≈ 100–1000 nm/s。
  换算下来**我们快了约 10⁶（一百万）倍**。
- 后果：拽太快时系统进入**"摩擦主导 / 远离平衡"**状态。此时断裂力主要来自
  **黏滞拖拽 + 原子硬挤（drag + steric friction）**，而**不是**真正区分 2B/WT/2M 的那点
  **能垒差异**。
- 有定律为证（Bell–Evans）：断裂力 ∝ ln(加载速率)。**速率拉到极端，所有东西都在高力下断，
  内在差异被淹没。** 这也是为什么 WT 都到了 1047 pN（实验仅 10–20 pN）。

> 你之前反复问的 **"1000 pN vs 文献 10 pN"** 的根子就在这里——不是参数填错，
> 而是拉伸速率本身使断裂力虚高了约两个数量级。

### 根因②（次因）：用错了突变类型——我们加体积，文献减电荷

- 2B 的真实机制是**"去掉一个盐桥"**（带电吸引消失 → 锁松）。文献的 2B 对照都用
  **减电荷型**突变：R1306**Q**、R1341**Q**、H1268**D**——把带电残基换成不带电、体积相近的，
  **干净地只破坏盐桥**。
- 我们用的是 R1306**W**（色氨酸）——**又大又占地方**。它在拉伸时制造**额外的原子挤撞/摩擦**，
  反而让 2B 断裂力**升高**。于是"丢盐桥（真 2B 机制）"和"塞了个大块头（纯人工假象）"
  **被混在一起**，信号被污染。

### 其它（背景噪声，非决定性）

- n 太小（单条件 5 reps）、早期结构预处理问题（Boltz D'D3–A1 的能量最小化一度炸到 Fmax 5.9e9）。
  这些是雪上加霜，**即使全修好也救不了①②**。

**小结：核心是①拽太快**（系统性地把信号埋掉），②用错突变是帮凶（让 2B 反成最高）。
两者叠加 → 完美反向 → SMD 这条路 no-go。

## 四、当前方案：平衡态 AIM–A1 盐桥 MD

既然 SMD 要做对必须**拽慢一百万倍**（算力上几乎不可行）且**还得换减电荷突变重跑**，
我们改用平衡态代理：

- **起点用实验结构 7A6O**（AIM-A1 的 X 射线晶体结构，2.12 Å）——比 de novo 预测结构可靠。
- **不拽**，只在体温下跑 ~50 ns，观察两个关键**自抑制盐桥是否保持**：
  - **D1269 ↔ R1306**（N-AIM ↔ α1 螺旋）
  - **D1269 ↔ R1450**（N-AIM ↔ α6 螺旋）
- 实测参考集：WT 这俩盐桥占有率高（如 D1269-R1306 ≈ 0.67），2B/2M 明显更低 →
  **"锁松没松"可量化** = `aim_sb_retained_z`。

> 重要诚实声明：平衡态盐桥**不是 2B 的正向判据**（2B 是力学现象，平衡态原则上看不见）。
> 它是 **2M 的旁证** + **2B 的"保护性"守门特征**（结合面保留 → 不像 2M），
> 在分类器里以**联合判据**形式接入，不单独硬判 2B。

## 五、可运行的流水线（A40 服务器）

新写了一个端到端驱动 `scripts/pipeline/run_new_2b_saltbridge_md.sh`，把已有的各阶段脚本串起来，
对 36 个 A1 域新 2B 变体跑平衡盐桥 MD（GPU 4–7）。四个阶段，各自可断点续跑：

| 阶段 | 做什么 | 调用 |
|---|---|---|
| `build` | 在 7A6O WT 骨架上用 FoldX 引入单点突变 | `build_2b_mutants_foldx.py --detect-offset` |
| `relax` | pdb2gmx + 真空/溶剂化能量最小化（CPU） | `relax_autoinhib_structure.sh`（逐变体） |
| `md` | NVT → NPT → production（GPU 轮转 4 卡） | `run_7a6o_variant_direct.sh` |
| `extract` | 算盐桥占有率 + z-score | `extract_aim_saltbridge_features.py` |

**运行方式**：

```bash
# A40 上，GPU 4-7
export FOLDX=/path/to/foldx
bash scripts/pipeline/run_new_2b_saltbridge_md.sh --preflight      # 先自检
nohup bash scripts/pipeline/run_new_2b_saltbridge_md.sh \
      --stage all --gpu-ids 4,5,6,7 --ns 50 \
      > output/gromacs_md_autoinhib/new_2b_driver.log 2>&1 &

# 也可单跑某阶段： --stage build | relax | md | extract
```

**变体清单**：`output/new_2b_a1_variants.txt`（36 个 A1 域新 2B，native VWF 编号，含 2 个 2M
对照 R1374C/R1374H 用于 z-score 校准）。注：`M1303M` 为同义突变（疑源表笔误），FoldX 会跳过。

**产物**：`output/md_7a6o_saltbridge_features.csv`（含 `aim_sb_retained_z`），
再跑 `validate_new_2b_panel_recall.py` / `validate_2b_recall_saltbridge.py` 把这一列并进分类器，
重测 **non-hotspot 2B recall**（当前无 MD 时仅 33%，这正是要靠盐桥特征提升的缺口）。

## 六、为什么这一步重要（给汇报的落点）

之前在更大 2B 集（34 个 clean 新变体）上验证分类器：
- **hotspot 18/19 = 95%**（hotspot 先验稳健，可上生产）；
- **non-hotspot 仅 5/15 = 33%**，且 3 个真 2B 被误判 2M（C1272R/H1268Q/V1316M）。

定量发现：**剩余失败模式全是"需要 MD"的病例**——静态结构预测（iPTM）分不开 non-hotspot 2B 的
GOF 方向。**平衡盐桥 MD 是补这块缺口的唯一在做的路径。** 本次 A40 跑完，就能回答
"加入盐桥特征后 non-hotspot 2B recall 提升多少"。
